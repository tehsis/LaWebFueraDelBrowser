Apps

Aplicaciones, que normalmente se instalan desde un store (appstore, playstore, marketplace, etc) y que se pueden instalar independientemente en un dispositivo (pc, tablet, telefono) haciendo uso de los recursos del mismo (llamadas, gps, contactos, etc.)

Ventajas

  • Pueden acceder a los recursos del dispositivo
  • Conocen perfectamente el sistema sobre el que se ejecutan
  • Esto hace que, normalemnte, funcionen más rapido
  • SDKs con componentes reusables

Vamos a unir los dos mundos

Node.js

Nos permite ejecutar código js en un entorno dentro de nuestro sistema operativo.

Incluso incluye modulos para acceder a recursos del mismo (filesystem, procesos, etc)

Podemos escribir servidores gracias a su modulo de http

O aplicaciones de escritorio gracias a herramientas tales como node-webkit

O herramientas para automatizar tareas

O lo que se les ocurra que pueda correr en un SO

Cordova

...o Phonegap

Nos permite crear, mediante plugins aplicaciones que accedan a recursos nativos

Se instala a través de npm y provee una CLI para crear y administrar nuestro proyecto.

$npm install -g cordova

Ventajas

  • Soporte para múltiples plataformas (Android y iOS quizá los más difundidos)
  • Un mismo codebase para todas las plataformas.
  • Permite definir diferencias entre cada plataforma para ajustar la UX.
  • Se puede usar de forma embebida en otras aplicaciones nativas (junto con otros componentes nativos)

Desventajas

  • Sus capacidades estan atadas al entorno base (se actualizan menos frecuente que un browser)
  • Frustración ante efectos y widgets nativos
  • ¿Escucharon que los desarrolladores de Android se quejan de la fragmentación?

Devices API

Conjunto de APIs que nos permite acceder al hardware y/o recursos de los dispositivos.

Documentación en MDN

Anteriormente se llamaban WebAPI.
Permiten que nuestras apps interactuen con el "contenedor" (SO/dispositivo)

Veamos algunas

API de vibración

                        
    // Vibrar por un segundo (o mil milesimas)
    window.navigator.vibrate(1000);
    // Hagamos bailar nuestro telefono
    window.navigator.vibrate([200, 100, 200]);

API de bateria


    // Podemos saber el nivel de bateria del dispositivo
    console.log(window.navigator.battery.level)
    // O saber si esta enchufado
    consoe.log(window.navigator.charging);
